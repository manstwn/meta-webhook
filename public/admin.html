Webhook Forwarding & 10m Temporary Links Documentation

Configure your VPS instance to forward all incoming WhatsApp webhook payloads directly to your backend service in real-time, including secure, tamper-proof media download links that expire automatically after 10 minutes.

1. Configuration via .env

Add the FORWARD_URL variable inside your VPS .env file: 

FORWARD_URL=https://your-domain.com/api/incoming-whatsapp-webhook

3. Handling Expiring Media Links

          

            The tempMediaUrl property contains a cryptographically signed HMAC token pointing directly to the file served on the VPS. 
          

          
    
            
  • Stateless Security: No database check is performed to load the media link. It validates the signature using the configured token secret.
    • 
            
  • 10-Minute Limit: If the link is accessed after 10 minutes, Express will automatically reject the download with a 410 Gone / Link Expired code.
